One or more pairs of codes and values separated by colons ":". The code and value is separated by a semicolon ";".

The code is stored in the data. The value is optional if there is only one pair.

Often only one pair is used as a single check box. However, if more than one pair is defined then all the checked codes are stored as multivalued in the field if single valued or as subvalues if the field is multivalued itself.

Multiple options are arranged vertically onscreen unless the horizontal property is set to true

Client side form events

All functions must return something otherwise an error message will appear.

no input or output

function form_postinit()

no input or output

return true, or false to cancel the key pressed

return false to cancel read

Form properties

gparameters

gparameters is a useful holding place for various parameters designed to customise a form depending on how it is called.

gparameters can be loaded initially from the following places

1. The script in a form
2. The URL after the ? character is broken intopropertynameandpropertyvaluepairs.AAA=111&BBB=222
3. The dialog arguments that may be passed into a window if it is opened in script

gparameters.XXXXXX

Sets any field XXXXXX to the contents of gparameters.XXXXXX after postinit.

gparameters.confirm

For unbound forms only. Indicates that after the user presses the save button, they have to confirm before the data is processed.

See also gparameters.savemode which may contain the word 'CONFIRM'

gparameters.savemode

may contain the words:

CONFIRM which will ask the user to confirm before processing the form on Save

EXECUTE which will call the relevent module and process on the server (automatic if a standalone window not called from another)

CLOSE which will automatically close the form after processing (automatic if called from another window)

gparameters.key

Contains the initial record key for file forms.

Can be set anytime upto and including form_postinit()

gparameters.defaultrevstr

Contains the initial form data for data forms.

Can be set anytime upto and including form_postinit()

gparameters.documenttitle

Can be set anytime upto but excluding form_postinit()

gparameters.cancel=true

Indicate that Cancel button is required on data forms not called with dialogArguments (normally suppressed)

gparameters.savebuttonvalue="OK'

can be set anytime upto and including form_postint

gupdateonlymode

prevent create and delete, allow read and update

greadonlymode

prevent create, update and delete, allow read

gpreventcreation

allow read, update and delete but not creation

gpreventdeletion

allow create, read and update but not deletion
